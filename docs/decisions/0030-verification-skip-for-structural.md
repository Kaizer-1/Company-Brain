# ADR 0030 — Verification Skips the Citation Check for Event-less Structural Answers

## Status

Accepted

## Context

The agent's anti-hallucination guarantee (ADR 0025) is `verify_provenance`: every `[evt:UUID]`
in the answer must appear in the tool's provenance, and an answer with no citation fails.
Phase 4C's structural tools break an assumption baked into that check: some structural answers
have **no source event to cite**. "How many active decisions are there?" is answered by
`count(n)` — the fact is about the graph's *structure*, not about any Postgres event. There is
nothing to put in `[evt:UUID]`, and the Pydantic `AnswerWithCitations.citations` field
(`min_length=1`) would reject such an answer outright.

## Decision

**When the route is structural (`get_entity`, `neighbors`, `enumerate`, `aggregate`) AND the
tool returned no `available_event_ids`, skip the inline-citation check and mark the answer
verified with empty citations.** Synthesis for this case uses a dedicated prompt
(`synthesis_structural.txt`) and the `StructuralAnswer` schema (citations optional). Every
other path — including a structural tool that *did* return events — keeps the full ADR 0025
check.

## Why this preserves the grounding contract

The grounding thesis is "no fabricated facts reach the user." For an aggregate, the *fact is
produced by deterministic typed Cypher*, not generated by the LLM. The LLM's only job is to
phrase the count the tool computed; it adds nothing to fabricate. The guarantee is upheld a
different way: the number is as trustworthy as the query, and the query is tested. A structural
tool that *does* return events (most `enumerate`/`get_entity`/`neighbors` calls — nodes carry
`source_event_ids`) is held to the normal citation check, because there it *can* cite.

## Alternatives Considered

### Option A — Skip the check for event-less structural answers (chosen)

**Pros**: lets count/grouping answers exist; keeps the strict check everywhere it is
meaningful; the skip is narrow (structural route AND empty events) and logged.

**Cons**: one code path where `verified=True` is set without a citation match — must be
unmistakable in code and docs (it is: explicit branch + this ADR).

### Option B — Require a sentinel citation like `[evt:graph]`

**Pros**: keeps a uniform "always cites" rule.

**Cons**: a fake citation that resolves to nothing is worse than honestly having none; the
frontend would render a dead citation chip.

### Option C — Relax `AnswerWithCitations.citations` to allow empty globally

**Pros**: no new schema.

**Cons**: removes the first-line guard for *every* route, including KQ/search where an uncited
answer genuinely is a failure. Far too broad.

## Consequences

**Enables**: aggregate counts and other event-less structural answers; the verifier no longer
has to choose between rejecting a correct count and accepting a fabricated citation.

**Constrains**: the skip is keyed on `route ∈ STRUCTURAL_ROUTES and not available_event_ids`;
that predicate is duplicated in the graph routing edge, the streaming terminal check, and the
verifier — they must stay in sync (all reference `STRUCTURAL_ROUTES`).

**Locked into**: structural tools must return a `QueryResult` (a `value` key) so the routing
edge can distinguish a genuine structural result from a fallback-to-search empty result.

**At larger scale / in production**: one could attach lightweight provenance even to
aggregates (e.g. the set of node events behind the count) and drop the skip entirely; for a
portfolio demo the honest "this is grounded in structure, not an event" framing is clearer.

## Interview Defense

> "Some structural answers — a count, mainly — have no event to cite; the fact comes from
> deterministic Cypher, not the model. Forcing a citation there would mean either rejecting a
> correct count or inventing a fake citation, both worse. So verification skips the inline
> check only when the route is structural and the tool returned zero events, and synthesis
> uses a citation-free schema for that case. Everywhere a citation is possible — including
> structural lookups that return node events — the strict ADR 0025 check still runs."
