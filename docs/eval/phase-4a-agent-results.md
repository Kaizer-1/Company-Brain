## Discussion

> The per-question table above is machine-generated. This Discussion is hand-written for the
> 2026-06-05 run; numbers are stochastic (two LLM calls per question), so a re-run will shift
> them by a few points without changing the conclusions.

### What passed

- **Route accuracy = 1.000 (target ≥ 0.85).** All 30 questions routed to the expected tool,
  including the deliberately colloquial phrasings (q08 "conflicts with", q10 "pushing back
  on", q14 "what downstream things break"). The few-shot router prompt plus enum-constrained
  output is doing its job. With only 30 questions this is "no misroutes in this set", not
  "100% forever"; a larger set would surface boundary cases.
- **Refusal correctness = 1.000 (target ≥ 0.80).** All five out-of-scope questions (weather,
  translation, arithmetic, a poem, an action request) routed to `unknown` with zero citations.
  The read-only action-refusal (q30, "email the team and deploy") is the safety-relevant one
  and it held.
- **Citation overlap = 0.608 (target ≥ 0.50).** Jaccard between the agent's citations and the
  typed query's own provenance (the gold set). 0.61 means the synthesiser usually cites a
  correct *subset* — grounded but conservative, often citing the one or two events that most
  directly answer rather than the full provenance chain. Acceptable: every cited id is real
  (verification guarantees it); the metric penalises *under*-citation, not fabrication.
- **Provenance verification rate = 0.864 (target ≥ 0.80).** Of 22 synthesised answers, 19
  passed on the first try. Three needed a retry (q06, q08) or exhausted the budget (q10). No
  answer with a fabricated citation ever reached the response — the loop caught every one.

### What missed: latency

- **Mean latency = 6645 ms (target ≤ 4000 ms) — MISSED.** The target was flagged "generous"
  and was still missed. Causes, by impact:
  1. **Two sequential network LLM calls.** Routing then synthesis each take ~2–4 s round-trip
     to `claude-3.5-haiku` via OpenRouter. They are inherently sequential (synthesis needs the
     route), so the floor is ~their sum.
  2. **Retries triple synthesis time.** q06/q08 (1 retry) and q10 (3 attempts) run 10–17 s.
     The verification loop trades latency for grounding — deliberate.
  3. **One search outlier.** q21 hit 18 s; long-tail OpenRouter latency on the synthesis call.
  - The distribution is bimodal: `unknown` questions (one LLM call, no tool, no synthesis) run
    in **1.4–2.3 s**; full KQ/search answers in **5–9 s**; retry cases in **10–17 s**. The
    *median* is ~6.4 s. The 4 s target is only reachable for the single-call paths.
  - **Mitigations (Phase 4B):** stream synthesis token-by-token (cuts *perceived* latency to
    first-token ~2 s); cache routing for repeat questions; use a faster model for routing only.
    None change the architecture — synthesis is already the isolated last node.

### Failure mode: q10 exhausted the retry budget

q10 ("decisions still in force but that people are pushing back on lately") routed correctly
to KQ2 and the tool returned real contradiction provenance, but the synthesiser cited event
ids **not** in that provenance across all three attempts (jaccard 0.00,
`error=provenance_failed`). This is the *designed* terminal failure: rather than emit a
fabricated-looking grounded answer, the agent returns its best effort flagged with `error`.
One genuine hard failure in 30 (3.3%) is honest. The next lever is shrinking the candidate set
the KQ2 synthesiser sees (KQ2 returns many events, inflating the chance of an off-by-one
citation). Logged and left honest rather than tuned away.

### Cost

- **Mean cost = $0.00307 / question**, below the $0.005–$0.02 expected band. `unknown`
  questions cost ~$0.001 (one routing call); full answers ~$0.003; retry cases ~$0.005–$0.007.
  A thousand questions is about $3 at this corpus and model.
